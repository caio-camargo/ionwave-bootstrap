# Acquisition Source Deep Dive: Quality Score Methodology

## Overview
Framework for evaluating acquisition sources beyond CAC using retention, LTV, payback period, and contribution margin. Based on Bootstrap File 12 "Acquisition Source Deep Dive" methodology, enabling source-level P&L analysis and priority ranking.

## Beyond CAC: The Quality Score Problem [Confidence: A | Source: Bootstrap File 12, M30 alignment, unit economics frameworks | Date: 2026-02]

### Why CAC Alone is Insufficient

**Traditional Approach**: Rank acquisition sources by CAC (Customer Acquisition Cost)
- Source A: $30 CAC → "Best"
- Source B: $50 CAC → "Worse"

**Problem**: CAC doesn't account for customer quality (retention, repeat purchases, LTV).

**Real-World Example**:
```
Source A (Meta Broad Targeting): $30 CAC
  - 30-day retention: 20%
  - Avg orders per customer: 1.2
  - LTV: $60
  - LTV:CAC = 2.0x

Source B (YouTube Educational Content): $50 CAC
  - 30-day retention: 60%
  - Avg orders per customer: 2.5
  - LTV: $200
  - LTV:CAC = 4.0x

Conclusion: Source B has 67% higher CAC but 2x better LTV:CAC (better investment)
```

**Key Insight**: Cheapest customer ≠ best customer. Must evaluate quality, not just cost.

## Quality Score Dimensions [Confidence: A | Source: Bootstrap File 12, cohort analysis frameworks | Date: 2026-02]

### Dimension 1: Customer Acquisition Cost (CAC)

**Definition**: Total cost to acquire one customer from source.

**Calculation**:
```
CAC = Total Ad Spend (Source) / New Customers Acquired (Source)
```

**Example**:
```
Meta campaign spend: $5,000
New customers acquired: 100
CAC = $5,000 / 100 = $50
```

**Important**: Must isolate NEW customers (not repeat purchases from existing customers).

**Tracking Method**: Use first-order attribution (track customer's first purchase source via UTM parameters or platform attribution).

### Dimension 2: Lifetime Value (LTV)

**Definition**: Total revenue generated by average customer from source over their lifetime.

**Calculation**:
```
LTV = Avg Order Value × Avg Orders per Customer × Gross Margin %
```

**Example**:
```
Avg Order Value: $80
Avg Orders per Customer (first 180 days): 2.0
Gross Margin: 60%

LTV = $80 × 2.0 × 0.60 = $96
```

**Time Windows**:
- **30-Day LTV**: Revenue in first 30 days (early signal, limited predictive power)
- **90-Day LTV**: Revenue in first 90 days (better signal, most e-commerce uses this)
- **180-Day LTV**: Revenue in first 180 days (strong signal, but long wait for data)
- **Lifetime LTV**: Full customer lifetime (gold standard, but requires years of data)

**Recommendation**: Use 90-day LTV as primary (balance between signal quality and data availability).

### Dimension 3: Payback Period

**Definition**: How long until customer generates enough gross profit to recover CAC.

**Calculation**:
```
Payback Period (days) = CAC / (Daily Revenue per Customer × Gross Margin %)
```

**Simplified Calculation** (using first-order payback):
```
First-Order Payback = CAC / (Avg Order Value × Gross Margin %)

If First-Order Payback <1.0: Profitable on first purchase (ideal)
If First-Order Payback 1.0-2.0: Profitable by second purchase (acceptable)
If First-Order Payback >2.0: Requires 3+ purchases to recover CAC (risky)
```

**Example**:
```
Source A:
  CAC: $50
  Avg Order Value: $80
  Gross Margin: 60%

First-Order Payback = $50 / ($80 × 0.60) = $50 / $48 = 1.04

Interpretation: Requires 1.04 orders to recover CAC (breaks even slightly after first order)
```

**Business Implication**:
- **<1.0**: Can scale aggressively (profitable immediately)
- **1.0-1.5**: Moderate scaling (need some repeat purchases for profitability)
- **>2.0**: Slow scaling (high dependency on retention, cash flow risk)

### Dimension 4: Retention Rate

**Definition**: Percentage of customers who make repeat purchase within time window.

**Calculation**:
```
30-Day Retention = (Customers with 2+ orders within 30 days) / (Total new customers)
```

**Example**:
```
New customers from source: 100
Customers who purchased again within 30 days: 25

30-Day Retention = 25 / 100 = 25%
```

**Retention Benchmarks by Time Window** [Confidence: B | Source: E-commerce industry benchmarks 2024-2026 | Date: 2026-02]:
- **30-Day Retention**:
  - Excellent: >30%
  - Good: 20-30%
  - Average: 10-20%
  - Poor: <10%

- **90-Day Retention**:
  - Excellent: >50%
  - Good: 35-50%
  - Average: 20-35%
  - Poor: <20%

**Why It Matters**: High retention indicates product-market fit, customer satisfaction, and LTV potential.

### Dimension 5: Average Order Value (AOV)

**Definition**: Average dollar amount per order from customers acquired via source.

**Calculation**:
```
AOV (Source) = Total Revenue (Source) / Total Orders (Source)
```

**Example**:
```
Total revenue from Source A customers: $8,000
Total orders from Source A customers: 100

AOV = $8,000 / 100 = $80
```

**Source-Level Variance**: Different sources may have different AOV (e.g., YouTube educational content may attract customers who buy larger bundles vs TikTok impulse buyers).

**Why It Matters**: Higher AOV reduces CAC:AOV ratio (improves first-order profitability).

### Dimension 6: Contribution Margin per Customer

**Definition**: Gross profit generated by customer after deducting COGS, shipping, platform fees, and CAC.

**Calculation**:
```
Contribution Margin = (Revenue × Gross Margin %) - CAC - Avg Shipping Cost - Platform Fees
```

**Example**:
```
Revenue (first order): $80
Gross Margin: 60% (revenue minus COGS)
CAC: $50
Shipping Cost: $5
Platform Fees (Shopify, Stripe): $3

Contribution Margin = ($80 × 0.60) - $50 - $5 - $3 = $48 - $58 = -$10

Interpretation: Losing $10 per customer on first order (need repeat purchases to become profitable)
```

**Why It Matters**: This is the TRUE profitability metric (accounts for all direct costs).

**Positive Contribution Margin**: Scale aggressively (profitable per customer)
**Negative Contribution Margin**: Need retention to recover loss (or improve AOV/reduce CAC)

## Quality Score Framework [Confidence: A | Source: Bootstrap File 12 methodology | Date: 2026-02]

### Composite Quality Score

**Purpose**: Single metric combining all dimensions for easy source comparison.

**Formula** (weighted scoring):
```
Quality Score = (LTV:CAC Ratio × 40%) + (Retention Rate × 30%) + (Payback Period Score × 20%) + (Contribution Margin Score × 10%)
```

**Component Scoring**:

**LTV:CAC Ratio** (40% weight):
- 5 points: LTV:CAC >4.0x (excellent)
- 4 points: LTV:CAC 3.0-4.0x (good)
- 3 points: LTV:CAC 2.0-3.0x (acceptable)
- 2 points: LTV:CAC 1.5-2.0x (marginal)
- 1 point: LTV:CAC <1.5x (poor)

**Retention Rate** (30% weight, using 90-day retention):
- 5 points: >50% retention (excellent)
- 4 points: 35-50% retention (good)
- 3 points: 20-35% retention (acceptable)
- 2 points: 10-20% retention (marginal)
- 1 point: <10% retention (poor)

**Payback Period** (20% weight):
- 5 points: <30 days (excellent)
- 4 points: 30-60 days (good)
- 3 points: 60-90 days (acceptable)
- 2 points: 90-120 days (marginal)
- 1 point: >120 days (poor)

**Contribution Margin** (10% weight, first-order):
- 5 points: >$20 positive (excellent)
- 4 points: $10-20 positive (good)
- 3 points: $0-10 positive (acceptable)
- 2 points: -$10-0 (marginal)
- 1 point: <-$10 (poor)

**Example Calculation**:
```
Source A (Meta Broad):
  LTV:CAC: 2.5x → 3 points × 40% = 1.2
  90-Day Retention: 30% → 3 points × 30% = 0.9
  Payback Period: 45 days → 4 points × 20% = 0.8
  Contribution Margin: $5 → 3 points × 10% = 0.3

  Quality Score = 1.2 + 0.9 + 0.8 + 0.3 = 3.2 / 5.0 (64%)

Source B (YouTube Educational):
  LTV:CAC: 4.2x → 5 points × 40% = 2.0
  90-Day Retention: 55% → 5 points × 30% = 1.5
  Payback Period: 30 days → 5 points × 20% = 1.0
  Contribution Margin: $25 → 5 points × 10% = 0.5

  Quality Score = 2.0 + 1.5 + 1.0 + 0.5 = 5.0 / 5.0 (100%)

Conclusion: YouTube (100% Quality Score) significantly outperforms Meta Broad (64%) despite higher CAC
```

## Tracking Implementation [Confidence: A | Source: Analytics implementation best practices | Date: 2026-02]

### Required Data Infrastructure

**1. UTM Parameter Tracking** (Source Identification):
```
Meta Campaign: utm_source=facebook&utm_medium=paid&utm_campaign=broad_targeting
YouTube Campaign: utm_source=youtube&utm_medium=paid&utm_campaign=educational_content
TikTok Campaign: utm_source=tiktok&utm_medium=paid&utm_campaign=impulse_offer
```

**Store in Backend**: Save UTM parameters with each order (first-order source = customer acquisition source).

**2. Cohort Analysis Setup**:
- **Cohort Definition**: Group customers by acquisition source + month acquired
- **Example**: "Meta Broad - Feb 2026 Cohort" (all customers acquired via Meta Broad campaign in Feb 2026)

**3. Metrics to Track** (per cohort):
- **CAC**: Ad spend / New customers (from platform reporting)
- **Orders**: Total orders from cohort (from backend)
- **Revenue**: Total revenue from cohort (from backend)
- **AOV**: Revenue / Orders
- **Retention**: % of cohort with 2+ orders (at 30/60/90 days)
- **LTV**: Total revenue per customer at 30/60/90/180 days

**4. Dashboard/Reporting**:
- **Tool Options**: Google Sheets (simple), Looker Studio (free), Tableau (advanced), Northbeam (paid attribution tool)
- **Update Cadence**: Weekly (CAC, orders, revenue), Monthly (cohort analysis, LTV, retention)

### Cohort Analysis Example

**Cohort: Meta Broad - Feb 2026**

**Month 0 (Feb 2026)**:
- New Customers: 100
- Ad Spend: $5,000
- CAC: $50
- Orders: 100 (all first orders)
- Revenue: $8,000
- AOV: $80

**Month 1 (March 2026, +30 days)**:
- Repeat Orders: 25 (25% retention)
- Repeat Revenue: $2,000
- Cumulative Orders: 125
- Cumulative Revenue: $10,000
- 30-Day LTV: $100

**Month 2 (April 2026, +60 days)**:
- Additional Repeat Orders: 10
- Additional Revenue: $800
- Cumulative Orders: 135
- Cumulative Revenue: $10,800
- 60-Day LTV: $108

**Month 3 (May 2026, +90 days)**:
- Additional Repeat Orders: 8
- Additional Revenue: $640
- Cumulative Orders: 143
- Cumulative Revenue: $11,440
- 90-Day LTV: $114.40
- LTV:CAC: 2.29x

**Analysis**: Meta Broad cohort shows 25% 30-day retention, 90-day LTV of $114 (2.29x LTV:CAC ratio). Acceptable performance, continue at current budget.

## Source Optimization Strategies [Confidence: A | Source: Channel optimization frameworks | Date: 2026-02]

### Strategy 1: Scale High-Quality Sources

**Trigger**: Quality Score >80%, positive contribution margin, LTV:CAC >3.0x

**Action**:
- Increase budget 30-50% per month (faster than standard 20% Rule due to high quality)
- Allocate majority of new budget to this source (50-70%)
- Test horizontal expansion (new audiences, creatives, placements within source)

**Example**: YouTube Educational campaign has 100% Quality Score → scale from $1,000/month to $1,500/month (50% increase), allocate 60% of $500 new budget.

### Strategy 2: Optimize Moderate Sources

**Trigger**: Quality Score 50-80%, marginal contribution margin, LTV:CAC 2.0-3.0x

**Action**:
- Maintain or slow-grow budget (10-20% increases)
- Focus on improving efficiency (creative refresh, audience refinement, offer optimization)
- Test quality-improving tactics:
  - **Longer Sales Cycle**: Educational content, nurture sequences (may increase LTV)
  - **Higher AOV Incentives**: Bundles, tiered pricing (improve first-order profitability)
  - **Retention Mechanisms**: Email marketing, loyalty programs (increase repeat purchase rate)

**Example**: Meta Broad campaign has 64% Quality Score → maintain $2,000/month budget, test bundle offer to increase AOV from $80 to $100.

### Strategy 3: Fix or Kill Low-Quality Sources

**Trigger**: Quality Score <50%, negative contribution margin, LTV:CAC <1.5x

**Action**:
- **Option A: Diagnose and Fix** (2-4 weeks):
  - **Poor Retention**: Product not resonating with source audience → test different creative angle or targeting
  - **Low AOV**: Impulse buyers → test upsells, bundles, minimum order incentives
  - **High CAC**: Inefficient targeting/creative → test new audiences, creative refresh
- **Option B: Reduce Budget** (50% cut):
  - Continue testing at lower risk while diagnosing
- **Option C: Pause** (if fix attempts fail after 4 weeks):
  - Redirect budget to higher-quality sources

**Example**: TikTok Impulse campaign has 42% Quality Score, negative contribution margin → reduce budget from $1,000/month to $500/month, test educational creative (vs impulse) for 4 weeks. If no improvement, pause and redirect to YouTube.

### Strategy 4: Test New Source Hypothesis

**When**: Existing sources plateaued, looking for new high-quality sources

**Approach**:
- **Hypothesis**: "YouTube educational content attracts higher-intent, higher-LTV customers"
- **Test Budget**: 10-20% of total budget (e.g., $500/month if total budget $5,000/month)
- **Timeline**: 90 days (need sufficient time for cohort analysis, LTV tracking)
- **Success Criteria**: Quality Score >60%, positive contribution margin, LTV:CAC >2.5x

**Example**:
```
Month 1: Launch YouTube campaign, $500 budget, acquire 7 customers (CAC $71)
Month 2: Track 30-day retention (4/7 = 57% retention, strong signal)
Month 3: Track 90-day LTV ($180 LTV, 2.5x LTV:CAC) → SUCCESS → Scale to $1,000/month in Month 4
```

## Common Pitfalls [Confidence: A | Source: Analytics error patterns, optimization mistakes | Date: 2026-02]

### Pitfall 1: Ignoring Sample Size

**Mistake**: Evaluating source with only 10-20 customers (insufficient for statistical significance).

**Example**:
```
Source A: 10 customers, 90-day LTV $200, LTV:CAC 4.0x → "Amazing!"
Reality: Small sample, 2-3 high-value outliers skewing average. True LTV likely lower.
```

**Solution**: Require minimum 50-100 customers per source before making scaling decisions (statistically significant).

### Pitfall 2: Short-Term LTV Bias

**Mistake**: Using 30-day LTV only (misses repeat purchase value).

**Example**:
```
Source A: 30-day LTV $80, LTV:CAC 1.6x → "Marginal"
Source A: 90-day LTV $120, LTV:CAC 2.4x → "Good"

Conclusion: Would have paused source at 30 days, but it's actually profitable at 90 days
```

**Solution**: Use 90-day LTV minimum (or longer for high-ticket/B2B products with longer cycles).

### Pitfall 3: Not Accounting for Platform Fees

**Mistake**: Calculating contribution margin using revenue - COGS - CAC only (ignoring shipping, payment fees, platform fees).

**Example**:
```
Revenue: $80
COGS (40%): -$32
CAC: -$50
Simplified Contribution Margin: -$2 → "Barely negative"

Actual:
Revenue: $80
COGS: -$32
Shipping: -$5
Payment Fees (3%): -$2.40
Platform Fees (Shopify): -$1
CAC: -$50
True Contribution Margin: -$10.40 → "Significantly negative"
```

**Solution**: Include ALL variable costs in contribution margin calculation (use channel_profitability_matrix.md template).

### Pitfall 4: Attribution Window Misalignment

**Mistake**: Platform reports conversions in 7-day window, but backend tracks all-time (double-counting, misattribution).

**Example**:
```
Platform (7-day attribution): "100 conversions from Meta campaign"
Backend (all-time, first-order attribution): "80 customers acquired via Meta campaign"

Discrepancy: 20 conversions platform claims but backend attributes to different source (direct, organic)
```

**Solution**: Use backend first-order attribution as source of truth (platform reporting inflated by view-through, assisted conversions).
