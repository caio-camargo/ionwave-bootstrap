## 1. What Is The Trade?

The Trade is a complete system — a multi-layered machine — that connects:

```
Studio → Labor Chain → Operator → Running Business → Returns
```

It is **not** the spreadsheet. It is **not** any single document. The Trade is the repeatable system that takes a validated business opportunity and turns it into an operating company, compressing ~12-18 months of startup chaos into ~4-6 weeks of structured execution.

**IonWave** (electrolytes, D2C health brand) is the first instantiation. The Trade is the machine. Every future brand gets the same franchise-grade package, and the package improves with each instantiation.

### How the Trade differs from a franchise

| | Franchise | Trade |
|---|---|---|
| **What you get** | Proven brand + proven system | Proven system + NEW brand |
| **Upside** | Limited. Royalty forever. No equity | Full equity upside. Studio takes carry, not royalties |
| **Operator freedom** | Rigid playbook | Decision authority within bounds |
| **Replication** | One brand × 1000 locations | One system × different brands and categories |
| **Incentive alignment** | Franchisor profits from fees regardless | Studio profits from equity — only wins when Operator wins |

## 2. The Five-Layer Architecture

Each layer connects to the next. The Trade is the full stack.

### L1: Studio (Origination Engine)
- **Operated by:** Nilo / Studio 4
- **Produces:** Complete franchise-grade packages — ODD, contracts, playbooks, hiring systems, brand assets, governance
- **Receives:** Market intelligence, thesis validation, feedback from running businesses
- **Current grade:** A (ODD complete, role engineering complete, contracts ready)

### L2: Labor Chain (Delegation Depth System)
- **Operated by:** VA → CoS → Recruiter (sequential)
- **Produces:** A hired, onboarded, equipped Operator ready to execute
- **Receives:** Role-engineered JDs, assessment rubrics, hiring kits, scoring thresholds
- **Current grade:** A (hiring system complete, all templates ready)

### L3: Operator (Entrepreneur-in-a-Box)
- **Operated by:** Recruited Operator
- **Produces:** A running business — revenue, customers, brand, supply chain
- **Receives:** The franchise package (ODD, contracts, vendor intros, website, creative, onboarding)
- **Current grade:** B/C (designed but not yet built — product samples, website, ads all pending execution)

### L4: Compliance (Governance Layer)
- **Operated by:** Compliance Manager + CoS
- **Produces:** Grades, escalation triggers, performance reports, risk alerts
- **Receives:** Daily metrics, milestone reports, operating agreement compliance data
- **Current grade:** A- (designed, needs live Operator to monitor)

### L5: Learning (Compounding Layer)
- **Operated by:** Studio + Platform
- **Produces:** Updated templates, refined rubrics, better hiring criteria
- **Receives:** Post-mortems, failure analysis, patterns across all Trades
- **Current grade:** B (designed, needs first Trade completed to generate learnings)

## 3. Entity Structure and Capital Flow

### Who owns what

```
TRADE (Parent Holding Company)
│
├── Studio 4 (Operating Partner / GP)
│   • Builds ODDs
│   • Sources Operators
│   • Provides oversight
│   • Earns carry / management fee
│
├── TUP 1: IonWave (Subsidiary / SPV)
│   • Own LLC, bank account, P&L
│   • Operator runs day-to-day
│
├── TUP 2: [Next Venture]
│
├── TUP 3: [Next Venture]
│
└── Compliance Manager (Service Provider)
    • Independent assessor
    • Reports to Trade/investors
    • No equity in TUPs
```

### How capital moves

```
Investors ──→ Trade (at holding level)
               │
               ├──→ TUPs (capital allocation per stage, gated)
               │     │
               │     └──→ Operations (product, ads, team, fulfillment)
               │
               │  ┌──────────────────────────┐
               │  │    Returns flow back      │
               │  └──────────────────────────┘
               │
               ├──← TUPs (returns, exits, dividends)
               │
               ├──→ Investors (distributions)
               │
               └──→ Studio (carry, fees)
```

### Stage-gated capital unlocks

| Stage | Typical Allocation | Unlock Criteria | Who Approves |
|-------|-------------------|-----------------|--------------|
| Thesis / ODD | $5K | Thesis approved | Studio |
| Table Set | $10–25K | ODD complete, Operator signed | Studio |
| First Blood | $25–50K | Table Set verified complete | Studio |
| Winner Found | $50–200K | PMF signals hit | Studio + Trade |
| Scale | $200K+ | Winner criteria met | Trade board |

**Key principle:** Capital unlocks are gated by verified milestones, not timelines. No phase transitions without verification.

## 4. The Execution Phases

The Trade executes in five sequential phases. Each phase has an entry gate, specific tasks with owners, and a verified exit gate.

**Status states:** Not Started → In Progress → Done → **Verified**
> "Verified" means someone checked the output against verification criteria and it passed.

## 6. Critical Path and Dependencies

The critical path is the longest chain of dependencies. If anything on this path slips, everything downstream slips.

```
Nilo + Claude produce M0 thesis (W0)
  → Admin (VA) hired (W1)
    → Cold email starts (W2)
      → PM co-founder #1 hired (W3)
        → Entity formed + cap table (W3–4)
          → Cluster production begins (W4)
            → Site live with content (W5)
              → Expert reviews (W5–7)
                → Investor-ready package (W7)
                  → Active raise (W8–12)
                    → FUNDED (W12)
```

| M0 locked (W2) | All cluster production |

## 7. The TUP Lifecycle (How Individual Pieces Move)

Each piece of the Trade (each TUP — Trade Unit Project) moves through its own lifecycle:

| Phase | State | Who's Active | Exit Gate |
|-------|-------|-------------|-----------|
| 0. Pre-Imagination | TUP doesn't exist yet | Nilo + Claude | P-tab exists with scope, win conditions, dependencies |
| 1. Imagination (AI Draft) | Being authored | Nilo/PM + Claude | All content tabs first-drafted. Librarian confirms completeness. |
| 2. Internal Review | Draft QA | Librarian + Conductor | Structural completeness signed off. No broken cross-references. |
| 3. Expert Review | External grading | Expert Reviewer(s) | Grade ≥ B = locks. Grade ≤ C = remediation loop. |
| 4. Imagination Lock | Frozen reference | System | Content becomes read-only. Versioned. |
| 5. Implementation Extract | Tasks extracted | Conductor + PM | Implementation TUP created with tasks, owners, deadlines. |
| 6. Execution | Tasks being done | Operator + Team | All tasks complete OR kill criteria triggered. |
| 7. Post-Mortem | Complete | PM + Librarian | Lessons captured. Templates updated. |

**Phases 0–4** = Imagination (designing the business)
**Phases 5–7** = Implementation (running the business)
**Phase 4 (Lock)** = The bridge. Imagination becomes reference material for execution.

## 8. How to Read This as an Operator or VP

### If you're the Operator:
1. You receive the franchise package at Phase 1 entry (after the Labor Chain delivers you)
2. Your job starts at **Phase 1, Task 1.1**: form the entity, get the product, build the store, produce the creative
3. You own every task in Phases 1–4. The CoS and CM verify your work against the criteria in the rightmost column
4. You succeed when you hit **Graduation** (Phase 4.8): $100K MRR, positive margins, full team, no Studio dependency
5. Your capital unlocks are gated: you get more money as you hit verified milestones

### If you're the VP / Investor:
1. The Trade is a stage-gated investment vehicle. Capital unlocks only after verified milestones
2. **Phase 0** is Studio risk (building the team). Phase 0 complete = labor chain built, Operator hired
3. **Phase 1** is setup risk. Phase 1 complete = everything ready to sell
4. **Phase 2** is market risk. Phase 2 complete = customers exist, funnel works
5. **Phase 3** is unit economics risk. Phase 3 complete = profitable at test scale
6. **Phase 4** is scaling risk. Phase 4 complete = the business runs independently
7. The Compliance Manager (L4) provides independent monitoring — they report to the Trade/investors, not the Operator
8. Graduation means the business runs without Studio input. That's the end state.

### If you're the CoS:
1. You verify the Operator's work against the criteria in this document
2. You manage the labor chain below you (Recruiter, later team hires)
3. You are the bridge between Studio oversight and Operator execution
4. If a milestone fails verification, it doesn't advance — you are the quality gate
