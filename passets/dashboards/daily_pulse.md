### Data Pipeline (How This Dashboard Gets Populated)

**Option A: Google Apps Script (RECOMMENDED for Phase 1)**

**Why**: Free, reliable, no infrastructure, runs on schedule

**How it works**:
1. **Google Sheet** acts as the dashboard database
2. **Google Apps Script** runs daily at 6:00 AM EST
3. Script fetches data from:
   - Shopify API (revenue, orders, customers)
   - Meta Marketing API (ad spend, ROAS, CPA)
   - Bank API via Plaid (cash balance)
   - 3PL API (inventory levels)
4. Script writes data to Google Sheet cells
5. **This markdown file** pulls from Google Sheet (or operator manually updates from Sheet)

**Implementation**:
- Script template: `/passets/dashboards/scripts/daily_pulse_sync.gs`
- Auth tokens stored in: Google Apps Script Properties Service (encrypted)
- Error handling: Email operator if any API fails
- Backup: If API unavailable, script uses yesterday's value + note

**Option B: Zapier/Make (No-code alternative)**

**Why**: Visual workflow builder, easier for non-technical operators

**How it works**:
1. Zapier multi-step Zap runs daily at 6 AM
2. Fetches from Shopify, Meta, Bank, 3PL
3. Writes to Google Sheet
4. Sends Slack notification with summary

**Cost**: $20-50/mo (Zapier Pro plan)

**Option C: Custom Dashboard Tool (Phase 2+)**

**Tools**: Triple Whale, Polar Analytics, Northbeam
**Cost**: $200-500/mo
**Why wait**: Overkill for Phase 1. Use when revenue >$50K/mo.

## 🔄 Update Protocol

### Automated Updates (Preferred)
- **Frequency**: Daily at 6:00 AM EST
- **Method**: Google Apps Script OR Zapier
- **Failure Mode**: If script fails, operator receives email alert → manual update

### Manual Updates (Fallback)
- **Frequency**: Every morning, 8:00-8:15 AM
- **Time**: 5-10 minutes
- **Steps**:
  1. Open Shopify app → Check yesterday's revenue, orders, AOV, new customers
  2. Open Meta Ads app → Check yesterday's spend, ROAS, CPA, CTR
  3. Open Bank app → Check current cash balance
  4. Calculate: Runway = Cash Balance / 7-day avg burn rate
  5. Check: Support inbox (unresolved tickets >24 hrs?)
  6. Check: 3PL dashboard (inventory days remaining)
  7. Update this file OR Google Sheet with values
  8. Identify: Any 🔴 Red status? → Set "One Thing to Fix Today"

## 🚦 Status Thresholds (When Metrics Turn Red)

### Revenue & Orders
- 🟢 Green: Within ±15% of 7-day average
- 🟡 Yellow: -15% to -30% vs 7-day average
- 🔴 Red: -30%+ vs 7-day average OR <$100 revenue in a day

### Ad Performance
- **ROAS**:
  - 🟢 Green: >2.5x
  - 🟡 Yellow: 1.5-2.5x
  - 🔴 Red: <1.5x
- **CPA**:
  - 🟢 Green: <$45 (target from M3 unit economics)
  - 🟡 Yellow: $45-60
  - 🔴 Red: >$60
- **CTR**:
  - 🟢 Green: >1.0%
  - 🟡 Yellow: 0.5-1.0%
  - 🔴 Red: <0.5% (creative fatigue or poor targeting)

### Cash & Runway
- 🟢 Green: >60 days runway
- 🟡 Yellow: 30-60 days (start planning: fundraise, cut costs, or scale revenue)
- 🔴 Red: <30 days (CRITICAL - immediate action required)

### Fires
- **Stockout Risk**: <7 days inventory at current sell rate = 🔴 Red
- **Support Backlog**: >10 tickets unresolved >24 hrs = 🟡 Yellow, >20 = 🔴 Red
- **Ad Account**: Spend paused OR frequency >5 = 🔴 Red
- **3PL Delay**: >10% of orders not shipped within 2 business days = 🔴 Red
